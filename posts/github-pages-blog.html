GitHub Pages 个人博客搭建记录

这是一篇示例文章，用于说明本站采用 GitHub Pages 的原因、访客端和管理端的区别，以及后续文章如何发布。

联系作者

为什么选择 GitHub Pages

当前阶段更需要一个低成本、易维护、能公开展示研究方向和学生项目实践的站点。GitHub Pages 适合托管静态网页，不需要购买服务器，也便于后续把项目代码、文档和博客内容放在同一个工作流中管理。

访客端与管理端

访客端就是大家在浏览器里看到的公开页面，包括首页、研究方向、项目与竞赛、学生实践机会、技能路线和博客笔记。管理端不做网页登录后台，而是通过 GitHub 仓库维护：修改页面、提交代码，然后由 GitHub Pages 自动发布。

后续如何发布文章

新增文章时，可以复制本页面作为模板，放到 `posts/` 目录中，再到 `notes.html` 增加文章列表入口。这样既保持结构简单，也方便后续迁移到 Jekyll、Astro 或其他博客框架。

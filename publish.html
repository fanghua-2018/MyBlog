Publishing Workflow

博客管理与发布流程

本站没有网页登录后台。访客端由 GitHub Pages 托管，站长管理端就是 GitHub 仓库的编辑、提交和发布流程。

Roles

权限边界

访客端

学生、同行和合作方只能浏览公开页面、复制链接、查看联系方式和进入 GitHub 主页。

站长端

李帅兵本人通过 GitHub 仓库修改 HTML、CSS、文章和项目内容，仓库权限决定谁能发布。

为什么不做网页登录后台

GitHub Pages 是静态托管服务，没有数据库和服务端登录能力。第一版采用 GitHub 仓库管理，稳定、透明、成本低。

Steps

新增博客文章步骤

01

复制文章模板

在 `posts/` 目录中复制一篇已有文章，改成新的英文文件名，例如 `remote-sensing-intro.html`。

02

修改文章内容

更新标题、日期、分类、正文、返回链接和复制链接按钮中的文章地址。

03

更新博客列表

在 `notes.html` 的文章列表中新增一条记录，包含标题、日期、分类、摘要和阅读全文链接。

04

提交到 GitHub

将修改后的文件提交到 GitHub 仓库。GitHub Pages 会根据仓库设置自动发布。
